Postman API自动化01--快速开始

Posted on 2019-05-01 | In API自动化

环境信息

目标

能自己动手完成最常见的Get和Post请求脚本；

预览

Postman工具主界面，了解各块的主要功能；

准备工作

1、Workspace的选择有几种方式：在默认Personal的My Workspace直接开始，在默认Personal下新建Workspace，在Team下新建Workspace；

2、新建Collection（Collection相当于一个项目）；

3、在Collection下配置文件夹（文件夹相当于模块），文件夹下可以添加子文件夹，还可以快速复制某一个文件夹；

Post请求

一个Post请求主要包括几个部分：
1、请求类型和请求Url：我们新建Post请求所以类型选择Post，同时填入请求的Url，这里固定的部分我们取了一个环境变量Url的值；
2、Headers：请求头，请求头可以携带很多数据，Postman中请求头可以设置模板，方便快速选择；

如果确认请求体、Url都没错，但请求返回一直不对，记得检查Headers中的参数设置是否正确；


3、Body：请求体，我们请求的参数和值都在这里设置，环境变量和全局变量在这里被大量使用；

如何获得请求体，在另一篇《常用辅助工具》中有介绍；
示例请求体中引用了两个环境变量username和password；

4、Test(可选)：提取返回中参数、结果断言，提取参数在接口测试场景中大多会用到，通过变量传递关键参数；
5、保存、执行请求；

Get请求

一个Get请求主要包括几个部分：
1、Params(可选)：请求参数，这里填写的参数会体现在Url中
2、Headers：请求头，正常业务请求中不可或缺；
3、Test(可选)：提取返回中参数、结果断言，提取参数在接口测试场景中大多会用到，用于传递关键参数；
4、保存、执行请求；

执行方式

执行方式分为单个执行和批量执行；
单个执行，直接在编辑请求页面右侧点击【Send】即可;

批量执行，在Collection右上角或主界面左上角的【Runner】均可以发起，批量执行时可以执行整个Collection，也可以选择子文件夹执行；

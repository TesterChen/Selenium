环境信息

目标

熟悉【Test】模块常用脚本，能设置环境变量、全局变量，完成对返回数据的转化，获取返回中特定数据并检查；
注意：所有的引号都应该是英文状态；

数据

后续我们要用到的返回样例数据；

返回数据转化

请求完成后，服务器返回的数据可以是xml或json字符串，我们拿到返回后需要对内容进行转换，转换成方便处理的json对象；

获取特定字段

我们往往需要获取返回数据中的某些重要字段，如：id、msg，我们前面已经拿到了返回数据的json对象，那么常用的脚本是这样的；

设置动态变量

动态变量我们可以设置两种：环境变量和全局变量；

环境变量和全局变量的存在是非常重要的，通过动态变量拼接接口场景；
两者的差异，主要是作用范围不一样，可以先将环境变理理解为局部变量，具体的差异在其他篇介绍；

变量的使用

环境变量和全局变量在使用上没有差异，都是通过两个花括号调用变量；

断言

断言是接口自动化中非常重要的部分，接口执行成功并不代表预期数据是正确的，这就需要我们对返回数据进行断言；
断言包括两大部分：请求状态相关信息，请求数据相关信息

日志输出

在脚本出现问题需要调试时，日志输出显得尤为重要，要输出日志我们需要做两步：在Test区域输出日志，打开日志输出控制台；
Step 1、在Test区域输出日志代码
// 打印上面获取的jsonObject
console.log(“jsonObject”, jsonObject);

Step 2、打开控制台日志显示
在Postman主页面左下角找到控制台日志输出图标

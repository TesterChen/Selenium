Postman API自动化07--通用断言设置

Posted on 2019-06-14 | In API自动化

环境信息

目标

更好更高效地使用Postman工具；

通用断言

我们通常会有这样的需求，我们在做接口测试时需要对所有请求都做某一个相同断言，有什么好办法？
有的！Postman在Collection和Folder都支持多方面的通用配置；
下图显示的是Collection的断言设置入口；

Collection的通用配置包括几大部分，我们常用的有前置处理脚本(Pre-request Script)、后置数据处理(Tests)，如断言脚本，变量设置(Varibles)；
下图中我们对这个Collection设置了通用的响应码断言；

Folder的通用配置相比Collection少了变量设置(Varibles)模块；

断言影响统计

我们来做个试验，我们先注释掉通用的断言(此时登录脚本Test中是有数据提取脚本的），执行脚本，查看执行结果；
执行结果显示PASSED数量为0，FAILED为0；

接下来，我们放开Collection中对响应码的通用断言，执行脚本，看执行结果；
执行结果显示PASSED数量为2，FAILED为0；

通过上面两次的执行，我们可以得到以下结论：
1、我们可以通过设置通用断言的方式来对特定范围的请求进行结果检查；
2、执行结果成功或失败，只有在存在断言的情况下才会计数；

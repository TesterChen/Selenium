Postman API自动化--02常用辅助工具

Posted on 2019-05-01 | In API自动化

目标

通过常见工具提升测试脚本设计效率；

获取请求体

获取请求体常见有两种方式：页面操作、后台日志；

页面操作：
1、页面操作是最直接的方式，而且有效地避免了参数张冠李戴的情况；
2、没有其他帮助的情况下可以通过这种方式进行接口测试；
3、在Chrome浏览器下按下F12或通过页面右键选择检查后切换到Network，再操作相关业务；
4、操作完成后，根据API相关文档找到对应关键字，即可通过搜索查看到相应请求，包括请求url和请求参数体；
点击view source可以直接复制json字符串；

后台日志：
通过查询后台日志拿到请求体数据，再丰富参数即可；

格式化json字符串

通过页面或日志拿到的json字符串往往是没有格式化的，需要我们再手动格式化，这时候我们需要借助工具；
快速格式化json字符串：http://www.bejson.com/

jsonPath校验工具

在提取返回数据的参数值时，需要用到jsonPath，简单的jsonPath可以直接写，但复杂的我们要一次性写对就比较困难了；
在线书写和校验jsonPath：http://jsonpath.com/

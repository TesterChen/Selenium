Postman API自动化08--运行时变量

Posted on 2019-06-14 | In API自动化

环境信息

目标

更方便地调试接口测试脚本；

单个调试

我有多个接口要调试，有些接口之间有依赖关系，这种情况我应该怎么办？
没问题！Postman在多方面均解决了这个问题；

直接通过请求编辑页的【Send】按钮执行请求，此时默认会更新并记录你的变量值（环境变量和全局变量），调试下一个接口时可以直接取用；

批量执行

通过【Runner】执行时，我们勾选【Keep variable values】，此时也会更新并记录你的变量值（环境变量和全局变量）；

如何确定变量

在哪儿可以看到我的值有没有被记录或更新？预览变量中可以直接看到，我们需要关注的是CURRENT VALUE；

那我怎么确定这个值是不是更新的呢？
我们可以编辑对应变量的CURRENT VALUE列，把它清除后再执行脚本，执行完成后，查看对应变量的CURRENT VALUE值；
